# План проекта: Mind Space

## Описание
**Mind Space** — трекер настроения для ежедневного отслеживания эмоционального состояния, выявления паттернов и поддержки ментального благополучия.

## Лицензия
Проект распространяется по лицензии **OpenSphere License**.

## Цели MVP
1. Быстрый ежедневный лог настроения (1–2 действия).
2. История записей по дням/неделям.
3. Базовая аналитика: среднее настроение, тренд, частые теги.
4. Напоминания о записи (опционально).
5. Экспорт данных пользователя.

## Основные пользовательские сценарии
- Пользователь добавляет запись: оценка настроения, заметка, теги.
- Пользователь просматривает календарь настроения.
- Пользователь открывает экран аналитики и видит тренд.
- Пользователь редактирует/удаляет запись.

## Функциональные требования
- Аутентификация (email/пароль или OAuth) — опционально для MVP.
- Создание, чтение, обновление, удаление записей настроения.
- Категории/теги эмоций (например: радость, тревога, усталость).
- Графики по периодам: 7/30/90 дней.
- Фильтрация по тегам и диапазону дат.

## Нефункциональные требования
- Простой и спокойный UI, доступность (контраст, размеры шрифтов).
- Быстрая работа на мобильных устройствах.
- Безопасность персональных данных.
- Возможность локализации (RU/EN).

## Предлагаемый стек
- Frontend: React + TypeScript.
- Backend: Node.js (Express/Fastify) + TypeScript.
- База данных: PostgreSQL.
- Графики: Recharts/Chart.js.
- Авторизация: JWT + refresh tokens.

## Модель данных (черновик)
- **User**: id, email, password_hash, created_at.
- **MoodEntry**: id, user_id, score(1..10), note, created_at, updated_at.
- **Tag**: id, name.
- **MoodEntryTag**: mood_entry_id, tag_id.

## Этапы реализации
1. **Инициализация**
   - Создать монорепозиторий/структуру `client` и `server`.
   - Настроить линтеры, форматтер, базовый CI.
2. **Backend MVP**
   - CRUD API для записей настроения.
   - Фильтрация по датам и тегам.
   - Миграции БД.
3. **Frontend MVP**
   - Экран добавления записи.
   - Календарь/лента записей.
   - Экран аналитики с графиком тренда.
4. **Качество и релиз**
   - Unit/integration тесты.
   - Подготовка Docker-конфигурации.
   - Документация по запуску.

## Бэклог после MVP
- Push-уведомления и гибкие напоминания.
- ML-рекомендации по самопомощи.
- Импорт из Apple Health / Google Fit (если применимо).
- Режим приватности с локальным шифрованием записей.

## Риски и смягчение
- **Низкая регулярность записей** → мягкие напоминания и геймификация streak.
- **Конфиденциальность данных** → минимизация данных, шифрование, прозрачная политика.
- **Сложность аналитики для пользователя** → короткие подсказки и простой язык.
